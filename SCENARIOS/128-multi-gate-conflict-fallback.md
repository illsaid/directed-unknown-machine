# 128 — Keep independent fallback gates independently evidenced

## Type

real decision handoff

## User

A quality lead delegating a temperature-excursion disposition for a refrigerated shipment.

## Situation

Temperature evidence remains conflicting. The supplied quarantine fallback requires both an independent assay slot within 48 hours and qualified cold-storage capacity for the full assay period. Separate confirmations establish the two operational gates.

## Input

Decision: Decide whether to release, quarantine, or destroy refrigerated shipment L-207.
Evidence: The carrier probe reports a maximum temperature of 7.8°C. The calibrated receiving logger reports 8.3°C. The tamper seal was intact at receipt. The independent laboratory confirmed in writing that it can begin the required stability assay within 48 hours. The receiving site separately confirmed qualified 2–8°C storage capacity for the full seven-day assay period.
Constraints: Release requires a supported maximum temperature at or below 8.0°C and an intact tamper seal. Quarantine is operationally permitted only when the named independent stability assay can begin within 48 hours and qualified 2–8°C storage is available for the full seven-day assay period. Do not use evidence for one operational gate to satisfy the other.
Success: Release if both release gates are satisfied. If the temperature gate is unresolved or the supplied temperature records remain conflicting, quarantine the shipment and order the named independent stability assay only if the laboratory can begin within 48 hours and qualified storage is available for the full assay period. Otherwise report that no supplied disposition branch is yet authorized.

## Expected useful outcome

The compiled contract must preserve both temperature records, mark the temperature gate unresolved, mark the seal gate satisfied, and recommend quarantine plus the named independent stability assay. The authorization trace must cite the laboratory confirmation for the assay-start gate and the receiving-site confirmation for the storage gate separately; neither confirmation may be treated as support for both operational gates.

## Actual outcome

Run 130 strengthened the recommendation obligation so evidence for one fallback-specific gate cannot be reused to satisfy a distinct fallback-specific gate unless the supplied evidence explicitly supports both.

## Whether the system helped

yes

## What broke

The prior obligation required every fallback-specific gate to be cited, but did not explicitly prevent one evidence item from being reused across distinct gates. A compact recommendation could therefore list both gates while grounding both in only the laboratory confirmation.

## What would make the result more useful

Test a fallback whose single supplied record explicitly supports two operational gates and verify that shared evidence is allowed only because its dual support is explicit.
