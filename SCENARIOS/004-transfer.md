# 004 — Transfer: nearby task-shaping use case

## Type

transfer

## User

A nontechnical project owner trying to turn messy notes into an agent-ready task.

## Situation

The user has a rough goal and wants to hand it to an autonomous coding agent, but the goal lacks a narrow success condition and could easily produce a dashboard or generic toolkit.

## Input

"I have a folder of messy text snippets from customer calls. Make something that helps me see what matters and what to do next."

## Expected useful outcome

The system should transform the vague goal into a narrow, testable task with a specific user, input, expected output, and kill criterion — without turning it into a generic dashboard.

## What would make the result more useful
